## The Default Constructor
A default constructor allows you to initialize a class without passing it parameters. An example:
```C++
class Point {
public:
    int x;
    int y;

    Point() = default;

    Point(int x_, int y_)
        : x(x_), y(y_) {}
};

// Default Constructor
Point p;

// Constructor with paramters
Point p{2, 3};
```

The only difference is that the default constructor allocates space for the Point but does not set x or y. If you try to access x or y, you will get garbage data. Conceptually, a constructor with parameters is the same as a default constructor and then assigning the x and y value.
```C++
// This
Point p;  
p.x = 2;  
p.y = 3;

// Is the same as this in machine code and performance
Point p{2, 3};
```

## Initializer Lists For Constructors
It is a different way of initiliazing member variables in the constructure of a class. The two examples are equivelant. The first one will do it with member wise assignments and the second example will use initializer lists.
```C++
Cylinder::Cylinder(double radius_param, double height_param) {
	base_radius = radius_param;
	height = height_param;
}
```

With initializer list.
```C++
Cylinder::Cylinder(double radius_param, double height_param)
	: base_radius(radius_param), height(height_param) {}
```

The benefits of initializer lists are that they avoid unecessary copies and sometimes they are the only way to initialize an object.

## Initializer Lists vs Member Wise Copy Initialization
Member wise copy initialization will create the instance of the class and then copy in the parameters to assign to member variables. This is an extra copy step that initialization lists avoid as the member variables are initialized at object creation. Another difference is that in initializer lists, the order of member variables does matter where as it doesn't in member wise copy.

This means that you will have to initialize your objects in the order they were declared in the class definition.

## Explicit Constructors
Adding the explicit keyword to a constructor makes it so the the compiler never uses it in an implicit conversion. In other words, if we have a class with a single parameter constructor of type double, we could pass a double into a function that requires that class. The compiler will see that you passed a double and that the constructor of the required class only has a single double parameter and implicitly create a class for you. That is not desired behaviour sometimes so to prevent this add the explicit keyword.
```C++
class Point {
public:
    int x;
    int y;

    explicit Point(int x_, int y_)
        : x(x_), y(y_) {}
};

// Default Constructor
Point p;
```
